# Product roadmap

Pobratym is being developed in stages. Safety gates take priority over feature count or release speed.

## Phase 0: transparent prototype

- [x] Mobile-first Ukrainian interface
- [x] Local supportive conversation scenarios
- [x] Visible crisis escalation controls
- [x] Verified national support links
- [x] Public safety and architecture documentation
- [ ] Automated checks for crisis routing
- [ ] WCAG 2.2 AA accessibility review

## Phase 1: reviewed conversation model

- [ ] Create a clinical advisory group
- [ ] Define supported and unsupported conversation categories
- [ ] Build an anonymized red-team evaluation set
- [ ] Review all crisis language with Ukrainian specialists
- [ ] Add graceful handling for mixed Ukrainian and Russian input
- [ ] Publish measurable safety acceptance criteria

## Phase 2: privacy-first backend

- [ ] Document the threat model and data flows
- [ ] Add encrypted transport and short-lived sessions
- [ ] Keep raw conversation retention disabled by default
- [ ] Separate safety telemetry from conversation content
- [ ] Add consent, deletion, and incident-response procedures
- [ ] Complete Ukrainian legal and privacy review

## Phase 3: human support network

- [ ] Build a continuously verified resource directory
- [ ] Support regional and role-specific referrals
- [ ] Define warm handoff to trained human responders
- [ ] Add operational coverage and fallback rules
- [ ] Establish partner response-time expectations

## Phase 4: limited pilot

- [ ] Conduct moderated usability studies
- [ ] Run an invitation-only pilot with qualified oversight
- [ ] Measure referral completion, false crisis alerts, and missed alerts
- [ ] Publish pilot findings, known limitations, and remediation work
- [ ] Obtain an independent safety review before broader access

## Non-goals

Pobratym must not:

- diagnose a mental health condition;
- recommend medication or treatment changes;
- present generated advice as professional medical guidance;
- replace emergency services or a qualified clinician;
- use engagement or conversation length as a success metric.

## How priorities are chosen

Roadmap items are prioritized by:

1. risk reduction;
2. access to verified human help;
3. privacy and user control;
4. accessibility and reliability;
5. additional product capability.
